技术学习周刊第 1 期

2018 年参与过 1 年的 ARTS 打卡，也因为打卡有幸加入了 MegaEase 能与皓哥（左耳朵耗子）共事。时过境迁，皓哥已经不在了，自己的学习梳理习惯也荒废了一段时间。

2024 年没给自己定具体的目标，只要求自己好好吃饭睡觉，只字不差的阅读、听讲。之前学习阅读的很多资料都没有好好沉淀，因此决定恢复之前每周打卡的习惯，参考阮一峰老师的技术爱好者周刊，每周将自己读过的好的文章、资料做整理分享。

  1. 40 questions to ask yourself every year

Obsidian 的 CEO 在 2016 年分享的每年都会问自己的 40 个问题以及站在 10 年维度上问自己的 40 个问题。新年伊始，不妨作为一份参考，回顾来路，展望未来。自己尝试回答了几个：

  • 你身边有人生孩子了吗？

有，我老婆给我生了个儿子。

  • 你身边有人去世了吗？

有，我 leader 以及最重要的职业导师左耳朵耗子猝然离世。

  • 今年你学到了什么宝贵的人生经验？

除了生死，都是擦伤。真正明白了乔布斯斯坦福演讲所说的：Because almost everything — all external expectations, all pride, all fear of embarrassment or failure - these things just fall away in the face of death, leaving only what is truly important.

  • 你最想培养的特质是什么？

一往无前的勇气，雷厉风行的活力

  • 你希望十年之后，什么依旧保持一致？

勤奋，好奇

  1. kubetools 库

一个收集记录了 Kubernetes 相关工具的库，看提交目前（截止到 2023.12）还挺活跃的，可以收藏起来，使用 K8s 的过程中觉得哪里不方便需要一些工具的话，可以到这个仓库里来看看。另外其还有一个官方网站 kubetools.io，有很多 blog 讲解了如何使用这些工具，非常有学习参考价值。我常用的看下来就两个：

  • kind: 本地快速启动 K8s 集群，非常适合做测试。
  • netshoot：网络调试工具，有时候在 Pod 里执行网络相关操作时会缺少很多工具，可以通过 netshoot 来解决。

  1. # Kubernetes v1.29 新特性一览
  2. # Kubernetes 1.29 全新特性： 抛弃 iptables 还在等什么…

1.29 是 Kubernetes 2023 年的最后一个大版本终于发布了，又可以升级踩坑了。之前滴滴故障据说就是 K8s 升级跨度过大导致的，K8s 为了解决容器的运维编排而生，但本身的运维又成了令人头大的问题，有种屠龙勇士变恶龙的既视感了。

  1. # 用 Easegress革新您的 Kubernetes 体验：Kubernetes Gateway API
  2. # 不改一行代码，通过 Easegress 增强 K8s Gateway API

之前每次有对流量处理的需求，都得去 Nginx Ingress Controller 看一大堆的注解文档来找有没有符合需要的。现在 K8s 终于为流量治理提供了一个标准的 API 来替代 Ingress 了。看了下 HTTPRoute 的使用方式，至少在负载均衡、路由方面做的还是挺全面的。公司的 Easegress 网关及时进行了适配，并做了进一步增强，将官网还未支持的限流、熔断等流量治理功能加入了进来。

Easegress 是皓哥生前和团队小伙伴一直在推进开发的网关，目前已经进入了 CNCF 沙盒，希望之后也会有更好的发展。
