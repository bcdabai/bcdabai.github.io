C++继承的相关访问归纳

举例三个类：公共（public）继承

class A                                class B :public A                            calss C:public B

{                                          {                                                  {

分别创建各自对象A a，B b，C c；

1、问：A类中的成员能访问到A中的那些成员？

理解题意：A类所以问的是类内访问

类内访问的权限public，protected、private。

2、问：A类的对象a可以访问到A的那些成员？

理解题意：A类的对象a访问属于是类外访问

处于保护protected和private成员对于类外是不可见的，所以只能访问到public成员。

3、问：B类的成员可以访问到B和A中的那些成员？

B类以public的继承方式继承了A，搞清楚public的继承方式规则：除了私有private不可继承外其余public、protected均可继承。

对于类内访问，凡是可继承就可以访问，所以B类内可以访问自己B中的public、protected、private成员和A中的public、protected成员。

4、问：B的对象b可以访问A、B的那些成员？

对于类外访问protected和private是不可见的，所以b只能访问到A的public和B的public。

以此类推C类内访问可以访问到A的public、protected和B中的public、protected以及自己C的public、protected和private；而对于类外对象c来说则只能访问到A的public、B的public、C的public。

总结：（1）对于类内访问，public、protected、private均可访问，而对于继承凡是可继承则在类内均可见；对于类外访问，则只能看到public部分。（2）public的继承规则，除私有成员private不可继承外，其余均可继承。

举例三个类：保护（protected）继承

首先了解保护继承的规则：

对于保护继承可以继承和自己权限相等和高于自己的，换句话说就是可继承public和protected，但高于自己的权限在派生类中要降级成和自己同等权限，而和自己同等级的不变；protected继承后从基类继承来的public再派生类中属于protected。（可能没看懂，没关系下面举例）

1、问：对于B类内访问可以访问到A和B的那些成员？

B从A中继承到了public和protected部分，但按照保护继承的规则从A继承来的public要降级成为protected权限，但类内访问均可看见，所以B类内访问可以访问A的public（在B中是protected）成员和A的protected成员，B类内可以访问自己类中的所有成员。

3、问：B的对象b可以访问到AB的那些成员？

b作为B的对象，属于类外访问，所以可以访问到B的public成员，而A以保护继承的方式继承到B其public成员到了派生类B中变成了protected成员所以对外不可见，自然b也就访问不到了。

2、对于C类，从B中继承到了A的public和protracted（在C中都是protected）继承了B的public和protected（在C中也都是protected）所以对于C的类内访问可以访问到A的public、protected和B的public、protected以及自己的所有成员。

根据上述类外访问以此类推出C的对象c只能访问到C的public部分。
