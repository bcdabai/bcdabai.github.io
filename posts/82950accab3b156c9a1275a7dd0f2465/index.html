前言

上一篇文章我们了解了动态路由中的rip那么本篇文章我们就来看看动态路由中的ospf，ospf的优势有无路由自环、路由变化收敛速度快、支持区域划分等优势因此它更加适用于大规模网络中。

一、目标

PC ping通 pC2

二、关键命令

# 设置ospf进程号可以写ospf 1或者直接ospf，如果直接写ospf它默认还是1，以下面演示为前者
ospf 进程号
#eg 
ospf 1
# 进入ospf区域
area 区域名
#eg 
area 0
# 宣告area区域网段
network 网段 + 反子网掩码
#eg
network 10.0.0.0 0.0.0.255

四、完整流程

1.根据拓扑图配置设备ip

【AR1】

# 进入系统视图
<AR1>system-view 
# 进入各个接口根据拓扑图配置ip
[AR1]interface g0/0/0
[AR1-GigabitEthernet0/0/0]ip address 10.0.0.1 24
[AR1-GigabitEthernet0/0/0]interface g0/0/1
[AR1-GigabitEthernet0/0/1]ip address 192.168.10.254 24

【AR2】

# 进入系统视图
<AR2>system-view 
# 进入各个接口根据拓扑图配置ip
[AR2]interface g0/0/0
[AR2-GigabitEthernet0/0/0]ip address 10.0.0.2 24
[AR2-GigabitEthernet0/0/0]interface g0/0/1
[AR2-GigabitEthernet0/0/1]ip address 192.168.20.254 24

【PC1】

【PC2】

2.配置ospf

区域规划

我们把骨干区域(蓝色)规划到area0，把10网段(红色)规划为area1，把20网段(紫色)规划为area2。如下图所示:

【AR1】

# 设置ospf进程号
[AR1]ospf 1
# 进入area 0区
[AR1-ospf-1]area 0
# 宣告area 0区网段
[AR1-ospf-1-area-0.0.0.0]network 10.0.0.0 0.0.0.255
# 进入area 1区
[AR1-ospf-1-area-0.0.0.0]area 1
# 宣告area 1区网段
[AR1-ospf-1-area-0.0.0.1]network 192.168.10.0 0.0.0.255

【AR2】

# 设置ospf进程号
[AR2]ospf 1
# 进入area 0区
[AR2-ospf-1]area 0
# 宣告area 0区网段
[AR2-ospf-1-area-0.0.0.0]network 10.0.0.0 0.0.0.255
# 进入area 2区
[AR2-ospf-1-area-0.0.0.0]area 2
# 宣告area 2区网段
[AR2-ospf-1-area-0.0.0.2]network 192.168.20.0 0.0.0.255

3.验证

验证路由器中是否存在路由表

【AR1】

# 查看AR1路由表
[AR1]display ip routing-table

【AR2】

# 查看AR2路由表
[AR2]display ip routing-table

可以看到AR1和AR2均存在一条Proto为ospf的路由下面就可以进行连通性测试了

4.连通性测试

ok那么此次的ospf到这里也就完结了。

总结

虽然ospf更加适用于大规模网络但是它相对于rip的配置来说没有难多少反而在我看来更加方便并且ospf更加稳定与高效。
