前言

Hybrid接口是华为设备特殊的二层接口模式，
可以类似于vlan和trunk接口，
可以对数据帧不打vlan标签和不打标签。

●实验目的

通过配置交换机SW1和SW2不同接口的access、trunk、hybrid属性，实现PC1能和PC2互通，PC2能和PC4互通，PC3能和PC4互通。

●需求分析

1.新建如下图所示拓扑图；

2.SW1中创建vlan、配置access、trunk和hybrid接口；
3.SW2中创建vlan、配置trunk和hybrid接口；
4.路由器R1和R2配置接口网关和直连路由的IP地址，静态路由。

●实验步骤

1.交换机SW1命令

vlan batch 10 20                     ##创建vlan10 20

interface Ethernet0/0/1
 port link-type access
 port default vlan 10                ##将e0/0/1接口划分到vlan10

interface Ethernet0/0/2
 port link-type access
 port default vlan 20                ##将e0/0/2接口划分到vlan20

interface GigabitEthernet0/0/1
 port link-type trunk
 port trunk allow-pass vlan 10 20
                                     ##将g0/0/1接口设为trunk口

interface GigabitEthernet0/0/2
 port hybrid pvid vlan 10            ##进g0/0/2接口贴上vlan10
 port hybrid untagged vlan 10 20     ##出g0/0/2接口脱去vlan10或vlan20

2.交换机SW2命令

vlan batch 10 20                       ##创建vlan10 20

interface Ethernet0/0/1
 port hybrid pvid vlan 10              ##进e0/0/1接口贴上vlan10标签
 port hybrid untagged vlan 10 20
                                       ##出e0/0/1接口脱去vlan10或vlan20标签

interface Ethernet0/0/2
 port hybrid pvid vlan 20              ##进e0/0/2接口贴上vlan20标签
 port hybrid untagged vlan 10 20
                                       ##出e0/0/2接口脱去vlan10或vlan20标签

interface GigabitEthernet0/0/1
 port link-type trunk
 port trunk allow-pass vlan 10 20
                                        ##将g0/0/1接口设为trunk口

interface GigabitEthernet0/0/2
 port hybrid pvid vlan 20               ##进g0/0/2接口贴上vlan20标签
 port hybrid untagged vlan 10 20        ##出g0/0/2接口脱去vlan10或vlan20标签

3.路由器R1命令

interface GigabitEthernet0/0/0
 ip address 192.168.10.1 255.255.255.0 
                                         ##配置网关IP地址

interface GigabitEthernet0/0/1
 ip address 11.0.0.1 255.255.255.252
                                         ##配置直连路由IP地址

ip route-static 192.168.20.0 255.255.255.0 11.0.0.2
                                         ##配置数据转发192.168.20.0/24网段的静态路由

4.路由器R2命令

interface GigabitEthernet0/0/0
 ip address 192.168.20.1 255.255.255.0 
                                          ##配置网关IP地址

interface GigabitEthernet0/0/1
 ip address 11.0.0.2 255.255.255.252
                                          ##配置直连路由IP地址

ip route-static 192.168.10.0 255.255.255.0 11.0.0.1
                                          ##配置数据转发192.168.10.0/24网段的静态路由

5.实验结果验证

PC1能够ping通PC2，PC3能够ping通PC4，PC2能够ping通PC4，实验成功。


●实验总结

配置hybrid口时，牢记心法口诀。
出口检查：查untag表 有标 脱 无标 查tag 有 放通 无丢掉
进口检查：查有无标签 有标 查tag表 有放通 无标 丢弃
无标 打PVID后 放通
PC1pingPC2数据传输过程：

  1. 数据从PC1到SW1的e0/0/1口进入交换机，贴上vlan10标签，源IP192.168.10.10，目标IP192.168.20.10 源MAC是PC1的，不知道目标MAC地址，进行ARP广播。
  2. 数据从SW1的g0/0/2接口出去，查untag表，有vlan10标签，脱去vlan10标签。
  3. 到达R1后，查看路由表，重新封装数据帧从g0/0/1接口转发出去。
  4. 到达R2后，查看路由表，重新封装数据帧从g0/0/0接口转发到SW2的g0/0/2接口，查看没有标签，打上pvid 20标签。
  5. 数据从交换机SW2的g0/0/1接口转发到SW1的g0/0/1接口，此过程一直贴有vlan20标签，
  6. 数据从SW1的e0/0/2接口出去时，脱去vlan20的标签，最终到达PC2。PC2单播回应PC1自己的MAC地址，两台主机就能通信了。

PC3pingPC4数据传输过程：
源IP192.168.10.11，目标IP192.168.10.12；源MAC
为PC3的，因为没有PC4的MAC地址，所以要进行ARP广播获取到PC4的MAC地址，
此过程中数据从PC3进入SW2的e0/0/1接口时，没有标签，打上pvid vlan10的标签，从e0/0/2接口出去的时候查看untag表，有vlan10标签，脱去vlan10标签后数据到达PC4。
PC2pingPC4数据传输过程：

  1. 数据从PC2到SW1的e0/0/2口进入交换机，贴上vlan20标签，源IP192.168.20.10，目标IP192.168.10.12， 源MAC是PC2的，不知道目标MAC地址，进行ARP广播。
  2. 数据从交换机SW1的g0/0/1接口转发到SW2的g0/0/1接口，此过程一直贴有vlan20标签。
  3. 从SW2的g0/0/2出接口时查untag表，有vlan20标签，脱去vlan20后到达R2的g0/0/0接口。
  4. R2查看路由表，重新封装数据帧从g0/0/1接口转发到R1的g0/0/1接口。
  5. R1查看路由表，重新封装数据帧从g0/0/0接口转发出去。
  6. 数据进入SW1的g0/0/2接口时没有标签，打上pvid vlan10的标签。
  7. 数据从交换机SW1的g0/0/1接口转发到SW2的g0/0/1接口，此过程一直贴有vlan10标签。
  8. 数据从SW2的g0/0/2接口转发出去时，查看untag表，有vlan10标签，脱去vlan10标签后到达PC4。PC4收到PC2的数据包后，单播回应自己的MAC地址，PC2和PC4就能互通了。
