g++编译错误“no matching function for call to..."的解决

2019独角兽企业重金招聘Python工程师标准>>>

如果在类中定义了和库函数同名的函数，如下面代码中的printf，那么再调用库函数printf的时候，就会有类似“no matching function for call to...”的编译错误，即使类中定义的函数和库函数参数、返回值并不相同。

#include <stdio.h>


class CAClass{
    public:
        CAClass() { printf("hello\n"); }
        ~CAClass() {}
    private:
        void printf() { }
};

int main()
{
    CAClass aClass;
    return 0;

}

解决办法是在包含库函数的头文件上加上namespace，并在调用的时候，加上命名空间，问题解决。代码如下：

namespace stdclib {
#include <stdio.h>
}

class CAClass{
    public:
        CAClass() { stdclib::printf("hello\n"); }
        ~CAClass() {}
    private:
        void printf() { }
};

int main()
{
    CAClass aClass;
    return 0;
}
