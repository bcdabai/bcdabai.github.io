MySQL学习（六）——SQL的语言规范

1. SQL的两个重要标准

SQL92 和SQL99

2. SQL分类

  1. 数据定义语言 DDL（Data Definition Languages）：对数据库结构的操作
    定义了不同的数据库、表、视图、索引等数据库对象，还可以用来增删改数据库和数据表结构

    主要的语句关键字：
    CREATE：创建数据库、创建数据表、创建视图等
    DROP：删除表
    ALTER：修改表（eg.给表增加一个字段或删除一个字段）
    RENAME：对表进行重命名
    TRUNCATE:清空表内容

  2. 数据库操作语言 DML （Data Manipulation Language）:对数据表记录的操作
    用于对增删改查数据库记录，并检查数据完整性
    主要的语句关键字：
    INSERT：添加一条记录
    DELETE：删除一条记录
    UPDATE：修改一条记录
    SELECT：查询记录

  3. 数据控制语言 DCL(Data Control Language)，控制对数据表记录操作的操作，事务
    用于定义数据库、表、字段、用户的访问权限和安全级别
    主要的语句关键字：
    GRANT：赋予权限
    REVOKE：回收相关权限
    COMMIT：提交记录
    ROLLBACK：撤销记录
    SAVEPOINT：回滚到具体的保存者上

3. SQL语言的规则与规范

3.1 基本规则

  1. SQL语句可以用一行或多行来编写，但为了提高可读性，各语句需分行写，必要时使用缩进
  2. 每条命令需使用结束标志符：;（\g）(\G)
  3. 关键字不能用缩写也不能用分行
  4. 所有小括号、单引号、双引号皆为成对出现
  5. 所有输入皆为英文半角字符输入模式
  6. 字符串型和日期时间类型的数据可以使用单引号表示
  7. 列的别名，尽量使用双引号，而且不建议省略as

3.2 SQL大小写规范（建议遵守）

  1. 在Windows环境下大小写不敏感
  2. 在Linux环境下大小写敏感
    数据库名、表名、表的别名、变量名等皆严格区分大小写
    关键字、函数名、列名、列的别名，忽略大小写
  3. 推荐采用统一的书写规范
    数据库名、表名、表别名、字段名、字段别名都是小写
    SQL关键字、函数名、绑定变量等都大写

3.3 注释

单行注释：#注释或-- 注释 --
多行注释：/* 注释 */

3.4 命名规则

  1. 数据库名、表名不超过30个字符，变量名不超过29个字符
  2. 只包含A-Z，a-z，0-9，共63个字符，不能有空格、保留字、数据库系统或常用方法冲突
  3. 同一个MySQL软件中，数据库不能同名
  4. 同一个数据库中，数据表不能重名
  5. 同一个数据表中，字段名不能重名
  6. 保持字段名和类型的一致性
