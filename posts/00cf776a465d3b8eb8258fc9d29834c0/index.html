idea控制台出现乱码的解决方案

概述：有时候控制台的关键说明出现乱码，就很令人烦恼

在 IntelliJ IDEA 中出现控制台乱码通常是由于编码设置不正确或者字体显示问题导致的。以下是一些可能的解决方案：

1. 设置项目编码

确保你的项目编码设置正确：

  1. 在 IntelliJ IDEA 中，点击顶部菜单中的 "File" -> "Settings"（或者使用快捷键 Ctrl + Alt + S）。
  2. 在弹出的窗口中，选择 "Editor" -> "File Encodings"。
  3. 确保 "Global Encoding"、"Project Encoding" 和 "Default encoding for properties files" 都设置为合适的编码，一般推荐使用 UTF-8。
  4. 勾选 "Transparent native-to-ascii conversion" 以确保正确的显示非 ASCII 字符。

2. 设置控制台编码

  1. 在 IntelliJ IDEA 中，点击顶部菜单中的 "File" -> "Settings"。
  2. 选择 "Editor" -> "Color Scheme"。
  3. 在右侧的窗格中，找到 "Console Font" 和 "Console Colors" 部分，确保字体和颜色设置正确。尤其是在 "Console Font" 中，选择一个支持你需要显示字符的字体。

3. 修改启动配置

如果是在运行程序时出现乱码，可以尝试修改运行配置：

  1. 在 IntelliJ IDEA 中，找到你的运行配置（通常位于顶部工具栏旁边的下拉菜单中）。
  2. 右键点击运行配置，选择 "Edit Configurations"。
  3. 在弹出的窗口中，找到 "Configuration" 部分，在 "VM options" 中添加以下参数：
-Dfile.encoding=UTF-8

4. 修改 IDE 编码

  1. 在 IntelliJ IDEA 中，点击顶部菜单中的 "File" -> "Settings"。
  2. 选择 "Appearance & Behavior" -> "System Settings" -> "Memory Settings"。
  3. 在 "VM options" 中添加以下参数：
-Dfile.encoding=UTF-8

5. 检查系统环境

确保你的操作系统的区域设置和语言设置是正确的。

6. 更新 IntelliJ IDEA

有时候，更新 IntelliJ IDEA 到最新版本也可以解决一些问题。
