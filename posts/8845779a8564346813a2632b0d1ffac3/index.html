【vue部署】本地正常，部署后报404

vu2本地运行正常，生成部署后报404

这是env.production 配置

//env.production
# 页面标题
VUE_APP_TITLE = 管理系统-生产环境

# 生产环境配置
ENV = 'production'

# 管理系统/生产环境
VUE_APP_BASE_API = '/api'

# 路由懒加载
VUE_CLI_BABEL_TRANSPILE_MODULES = true

这是env.development 配置

//env.development 
# 页面标题
VUE_APP_TITLE = 管理系统-开发环境

# 开发环境配置
ENV = 'development'

# 开发代理。!!!!! 在该文件的同级目录下新建`.env.development.local`文件配置。

VUE_APP_PROXY_URL=http://192.168.193.210:8088/fee

# 管理系统/开发环境
VUE_APP_BASE_API = '/api'

# 路由懒加载
VUE_CLI_BABEL_TRANSPILE_MODULES = true

以下是我的vue.config 代理配置

//vue.config
proxy: {
      [process.env.VUE_APP_BASE_API]: {
        target: process.env.VUE_APP_PROXY_URL,
        changeOrigin: true,
        pathRewrite: {
          ['^'+process.env.VUE_APP_BASE_API]: ''
        }
      }

##问题 ：以上配置，在本地是没问题的，但是上线后，请求接口就报404

##解答：假如请求接口是: /index
1，本地开发才会用到vue.config.js 里的代理配置，生产环境并不会使用这个代理
/api/index ,这里的api就是使用的vue.config.js里的代理
2，上线部署后，使用的是env.production里的配置， 如下代码 ，
VUE_APP_BASE_API变量拼接接口获取的完整请求地址 http://192.168.193.210:8088/fee/index

//env.production
VUE_APP_BASE_API = http://192.168.193.210:8088/fee
