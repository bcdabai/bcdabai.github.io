Windows激活

slmgr.vbs -upk   #卸载现有Key

# 或

slmgr /upk   #卸载密钥

ver | find “10.0.”>nul && slmgr.vbs -ipk NPPR9-FWDCX-D2C8J-H872K-2YT43    #输入新Key (Win 10 Ent的GLVK)

# 或

slmgr /ipk W269N-WFGWX-YVC9B-4J6C9-T83GX   #安装（指定）密钥，红色部份替换为相对应的系统版本密钥（这里为Windows10专业版密钥）

# 或

slmgr /ato   #执行激活

slmgr.vbs -dlv   #查询激活状态。

Windows使用vlmcsd部署KMS激活服务器

2023  https://www.landui.com/help/nshow-11208.html

资源：利用 vlmcsd 搭建 KMS 服务器，Oracle VM VirtualBox 版本

  https://download.csdn.net/download/ken2232/88405118

使用限制：在单机，或局域网内使用。
使用 7z解压缩。zip压缩格式，有时会被视为病毒而被删除。
在虚拟机中运行 KMS 服务器，可以避免病毒之类的困扰。


一、服务器端：

1. 目录结构：
KMS_VirtualBox_VMs_C_Disk_Root 需要放在 C盘的根目录。否则，需要重新创建 VBox (Oracle VM VirtualBox) 虚拟机。

2. 使用 VBox 打开虚拟机，启动虚拟机，并获得 VM 的 ip4 地址。
C:\KMS_VirtualBox_VMs_C_Disk_Root\kms-windows-office\kms-windows-office.vbox
如：192.168.1.12

二、Windows 端

1. 以管理员身份运行“命令提示符”终端
2. slmgr /skms 192.168.1.12    #KMS服务器IP） ：设置kms服务器的地址或域名
3. slmgr /ato   #执行激活
4. slmgr /xpr   #验证激活是否成功（查看激活后的到期时间）

三、再次激活时，密钥失效

1. 上网搜索对应版本的新密钥。
2. 执行卸载旧密钥，安装新密钥。
3. 执行 二、Windows 端 中的步骤。

四、非必须的详细步骤

工具下载：
https://github.com/Wind4/vlmcsd/releases/tag/svn1113

1. Use Oracle VM VirtualBox to open:
C:\KMS_VirtualBox_VMs_C_Disk_Root\kms-windows-office\kms-windows-office.vbox

2. Start VM VirtualBox
Get the VM ip4，Such as:
udhcpc:lease of 192.168.1.12 obtained, lwase time 3600

3. step
3.1. slmgr /upk   #卸载密钥
3.2. slmgr /ipk W269N-WFGWX-YVC9B-4J6C9-T83GX   #安装（指定）密钥，红色部份替换为相对应的系统版本密钥（这里为Windows10专业版密钥）
注意：3.1 /3.2 不是必须的；当第二次激活显示密钥失效时，才需要。

3.3. slmgr /skms 192.168.1.12    #KMS服务器IP） ：设置kms服务器的地址或域名

3.4. slmgr /ato   #执行激活

3.5. 查询激活状态。
slmgr.vbs -dlv   
# 或
slmgr /xpr   #验证激活是否成功（查看激活后的到期时间）
