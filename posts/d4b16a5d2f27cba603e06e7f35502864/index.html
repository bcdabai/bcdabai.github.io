pnpm install 报 ERR_PNPM_LINKING_FAILED Error: EPERM: operation not permitted, rename

  1. 通过 nvm 安装了某个版本的 node,
  2. 之后又通过 npm install pnpm -g 安装 pnpm 了, 此时 pnpm 版本号是 v8.7.5,
  3. 然后在项目中执行 pnpm install 报如下错误:
ERR_PNPM_LINKING_FAILED  Error: EPERM: operation not permitted, rename

然后自己查找解决方案, 包括网友们给的各种方案, 都没有解决我遇到的这个问题

可能我的这个问题产生的原因比较特殊吧, 后来不经意的一个念头想了想, 回退下 pnpm 的版本呢, 然后直接就成功呢

原因可能是 pnpm 版本太高, 校验太严格了导致的

总结–我的这个问题的解决方案是:

	降低 pnpm 的版本就好了

	npm install -g pnpm@8.6.2

降低版本后, 然后再 pnpm install , 我的项目就不报上面的错误了
