经过反复的查询和调试终于发现是配置的问题,配置文件中id名与控制层声明的接口名称相同即可。
消费者配置： 

<dubbo:reference id="jc" interface="com.MP.manager.JcManager" check="false" timeout="12000"/>

jc为控制层中定义的接口名

<pre name="code" class="java">@Scope("prototype")
@Controller
@RequestMapping("/jc")

public class jcCtrl {
	private JcManager jc;

	public JcManager getJc() {
		return jc;
	}
}
提供者配置： 
 
 
<pre name="code" class="java"> <!-- 声明需要暴露的服务接口 -->
    <dubbo:service interface="com.MP.manager.JcManager" ref="jc" />
    <!-- 和本地bean一样实现服务 -->
    <bean id="jc" class="com.MP.managerImpl.JcManagerImpl" />
