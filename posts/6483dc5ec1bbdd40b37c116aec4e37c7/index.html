[一] ASP.NET Core概述

ASP.NET Core是一个跨平台，高性能的开源框架。ASP.NET Core是基于 ASP.NET 4.x 系列（类似于Java的JSP时代）的重新设计的。

1、特性

（1）跨平台
ASP.NET Core应用程序可以托管在 IIS，Apache，Docker 甚至自己把自己托管在进程中(俗称：自托管)。

（2）统一的 MVC 和 WebAPI 的技术栈
在MVC Controller和ASP.NET Web API在这两种情况下，创建的Controller 都从相同的Controller基类继承并返回IActionResult.

（3）依赖注入
开箱即用，ASP.NET Core内置支持依赖注入。

（4）可测试性
通过内置的依赖注入和用于创建 Web 应用程序和 Web API 的统一编程模型，可以轻松地对 ASP.NET Core应用程序进行单元测试和集成测试。

2、创建项目

各个模板的声明：

空: "空"模板不包含任何内容，需要从头开始手动设置所有内容。

MVC: 此模板包含Model，视图(Views)和控制器（Controller）文件夹，并添加 Web 特定的内容，如 CSS，JavaScript 文件，布局文件和网站所需的其他资源，也可以基于此模板创建 Restful 风格的 HTTP 服务。

Web API: 此模板包含创建 ASP.NET Core RESTful HTTP 服务所需的一切，API 模板只会创建Controllers文件夹，不会创建 Models 和 Views文件。 API没有用户界面,所以它不需要网站特定的内容，如 JavaScript 文件，CSS 文件，视图文件，布局文件。Web API 公开的数据通常由其他应用程序使用,可以简单的理解为:程序和程序之间打交道。

项目启动文件解读：

1）Program.cs文件中的Main()方法是程序入口，它主要做一些项目启动前的配置工作。

2）Main()方法调用静态类WebHost中的静态方法CreateDefaultBuilder()。这个方法执行以下几个任务，如:

  • 将Kestrel 用作 Web 服务器并启用 IIS 集成。
  • 从种配置源中加载配置。
  • 配置日志记录

Startup 类虽然只有两个方法：

  • ConfigureServices()方法配置应用程序所需的服务
  • Configure()方法配置应用程序的请求处理管道
