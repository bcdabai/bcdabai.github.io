问题：

前几天生产工控机上启动SQL Server 的时候出现“Windows 不能在 本地计算机 启动 SQL Server (MSSQLSERVER)。有关更多信息，查阅系统事件日志。如果这是非 Microsoft 服务，请与服务厂商联系，并参考特定服务错误代码 3417”

解决方案

网上提供了很多解决方案，有的说将服务的账号改成本地账号、sql server 安装目录把“压缩以释放空间”选项勾选去掉都没解决问题，最后通过将另一个正常启动的数据库的master库的mdf和ldf文件拷贝覆盖掉就正常了，这证实了master数据库坏了。数据库安装路径根据自己数据库的版本和系统定。
