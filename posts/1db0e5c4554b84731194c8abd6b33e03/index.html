使用jersey组件向图片资源服务器上传图片报405 Method Not Allowed错误

报错信息大致如下:

按照B站教程学习springmvc框架
B站springmvc教程
照着老师思路配置和部署的.

环境:WIN7 IDEA tomcat9.0.6 jdk1.8
1.除了spring全家桶之外相关包引入:

    <!--文件上传组件-->

    <dependency>

      <artifactId>jersey-client</artifactId>
      <version>1.18.1</version>

    <!--用于跨服传文件(向文件服传输文件) 结束-->

4.相关controller:

@Controller
@RequestMapping("/user")
public class UserController {
@RequestMapping("/fileupload3")
	//fileupload1和2都是好使的这里不赘述了 只保留fileupload3
	@RequestMapping("/fileupload3")
	public String fileupload3(MultipartFile upload) throws Exception {
		System.out.println("fileupload3");
		//定义上传服务器的路径
		String path = "http://localhost:9090/uploads/";
		String filename = upload.getOriginalFilename();
		//把文件名称设置成唯一值, uuid
		String str = UUID.randomUUID().toString().replace("-", "");
		filename = str + "_" + filename;
		//创建客户端的对象
		Client client = Client.create();
		//和文件服务器进行链接
		WebResource webResource = client.resource(path +  filename);
		//上传文件
		webResource.put(upload.getBytes());
		return "success";
	}
}

这里再说明一下情况:
前面传统上传文件和springmvc上传是好使的 上传文件也都存在于相应的uploads文件夹下
第3种是新建另一个服务器作为存图片服 使用jersey组件把文件上传到新服上(新的服几乎没有任何配置)

网上百度405资料.有说post方式不对,我把webResource.put改成了post/get都不好使,应该不是这个问题
后来一篇文章提出 改一下tomcat权限.为我提供了新思路,于是百度tomcat相关配置.最后找到了另一篇文章
说的是在tomcat目录把只读关掉
相关配置(在tomcat/conf/web.xml中加):

		<init-param>
			<param-name>readonly</param-name>
			<param-value>false</param-value>
		</init-param>

当然最后还报了一个409的错误.
我在项目目录/target/项目目录/下加了个uploads文件夹就好使了
