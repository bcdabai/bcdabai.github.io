git使用pull 远程仓库到本地——清除本地修改

开发者 A 发布了一个版本v1
开发者 A 在自己本地修改了文件，没有提交commit没有satge缓存
开发者 B 在v1 基础上修改了，发布了 V1.1
开发者A 要同步到 V1.1

  1. 本地 working tree 下文件修改，没有进行satge和 commit
  2. 先 git reset 到最后一个 commit 的版本
  3. git cleanup ，清理掉不要的且未保存到本地库的本地文件
  4. pull 或 sync 远程，更新到远程最新版本
