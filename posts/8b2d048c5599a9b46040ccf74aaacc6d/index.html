响应式布局+bootstra

定义

  响应式布局需要一个父级作为布局容器，来配合子级元素来实现变化效果。

  原理就是在不同的屏幕下，通过媒体查询来改变这个布局容器的大小，在改变里面子元素的排列方式和大小，从 而实现不同屏幕下，看到的不同的布局和样式变化。

平时响应式尺寸的划分：

超小屏幕（手机，小于768px）: 设置宽度为100%

小屏幕（平板，大于768px）：设置宽度750px

中等屏幕（桌面显示器，大于等于992px）：宽度设置为970px

大屏幕（桌面显示器，大于等于1200px）:宽度设置为1170px

一般情况下这个布局容器的宽度是定死的。用媒体查询来改变档位的变化。

 .container {
            height: 150px;
            background-color: pink;
            margin: 0 auto;
        }
        /* 1.超小屏幕 小于768px布局容器的宽度为100%*/
        
        @media screen and (max-width:768px) {
            .container {
                width: 100%;
            }
        }
        /* 2.小屏幕 大于768px布局容器的宽度为100%*/
        
        @media screen and (min-width:768px) {
            .container {
                width: 750px;
            }
        }
        /* 3.小屏幕 大于992px布局容器的宽度为100%*/
        
        @media screen and (min-width:992px) {
            .container {
                width: 970px;
            }
        }
        /* 4.小屏幕 大于1200px布局容器的宽度为100%*/
        
        @media screen and (min-width:1200px) {
            .container {
                width: 1170px;
            }
        }
    </style>

自适应和响应式的区别：

响应式的案例：改变浏览器宽度，“布局”会随之变化，不是一成不变的，例如导航栏在大屏幕下是横排，在小屏幕下是竖排，在超小屏幕下隐藏为菜单，也就是说如果有足够的耐心，在每一种屏幕下都应该有合理的布局，完美的效果。优点：适应pc和移动端，如果足够耐心，效果完美缺点：要匹配足够多的屏幕大小，工作量不小，设计也需要多个版本。

rem布局：改变浏览器宽度，页面所有元素的高宽都等比例缩放，也就是大屏幕下导航是横的，小屏幕下还是横的只不过变小了。。优点：理想状态是所有屏幕的高宽比和最初的设计高宽比一样，或者相差不多，完美适应。缺点：碰到重视高度的设计，或者重视元素间间距的设计，那就玩不开了。

bootstrap布局容器
 

   bootstrap需要为页面内容和栅格系统包裹一个 .container容器，bootstrap预先定义好了一个类，叫.container,它提供了两个作此用处的类。
 

1.container类
 

在bootstrap里面他就规定了这4个档位，就不用再在页面内写媒体查询了。
 

但是要注意，他定义了左右15px的内边距。
 

​ 1.1用于响应式布局的容器 固定宽度
 

​ 1.2 大屏（>=1200PX）宽度固定为1170px
 

​ 1.3中屏（>=992px）宽度定位970px
 

​ 1.4小屏（>=768px）宽度定位750px
 

​ 1.5超小屏（100%）
 

2.container-fluid类
 

2.1 用于流式布局容器 百分百宽度（跟屏幕一样宽）
 

2.2 占据全部视口（viewport）的容器
 

2.3适合于单独做移动端开发（如果是做移动端，可以用这个）
 

2.4。注意，由于 padding 等属性的原因，这两种 容器类不能互相嵌套。
 

bootstrap的栅格系统
 

栅格系统英文为“ grid systems ”，也有人翻译为“网格系统”，它是指将页面布局划分为等宽的列，然后通过列数的定义来模式化模块化页面布局。
 

栅格系统是把页面内容划分为若干个等宽的列。rem的把整个屏幕划分为10等份。
 

bootstrap提供了一套响应式、移动设备优先的流式栅格系统，随着屏幕或视口（viewport）尺寸的增加，系统会自动分为最多12列。
 

bootstrap里面container宽度是固定的，但是不同屏幕下，container的宽度不同，我们再把container划分12等份。
 

总结：在bootstrap里把container划分为12等份。container宽度不同，每一列的宽度就不同，里面内容会随着每一列的宽度的变化而变化。
 

栅格系统的使用
 

​ 1.在栅格系统内他自动引入了Normalize.css初始化样式，所以以后就不用在引入了。
 

​ 2.栅格选项参数
 

​ 栅格系统用于通过一系列的行（row）和列（column）的组合来创建页面布局，你的内容就可以放入这些创建好的布局中。
 

​ 注意点：遇到通栏的盒子就不需要container盒子了。先用一个普通的行，然后行里面在套用container，类似于版心。
 

3.行（row）必须放在container布局容器里面。
 

4.实现列的平均划分，需要给列加类前缀
 

5.每一列默认左右有15 px的padding。
 

6.可以同时为一列指定多个设备的类名。以便划分不同份数，例如：class=" col-md-4 col-sm-6 "
 

7.如果孩子的份数相加等于12，则孩子的份数就能占满整个盒子的container的宽度。
 

如果孩子的份数相加小于12，则孩子的份数占不满container的宽度。container的宽度会有空白。
 

如果孩子的份数相加大于12，多出来放不开的会另起一行显示。
 

8.细节：
 

  1.  
    栅格系统是往上兼容的：意味着小屏幕上的效果在大屏幕上也是可以正常显示的，但是大屏幕上的设置在小屏幕上却无法正常显示。
     
  2.  
    row可以再次嵌套在列中。如果不能填满整列，则默认从左排列，如果超出，则换行展示
     
 

案例：
 

<div class="container">
        <div class="row">
            <!-- 实现的效果是：大屏下一个div占3份，一行显示。中屏下一个div占四份，一行三个，另外一个另起一行。小屏幕下每个div占6份，两行显示。超小屏幕下，一个div占12份，每个div一行显示。 -->
            <div class="col-lg-3 col-md-4 col-sm-6 col-xs-12">1</div>
            <div class="col-lg-3 col-md-4 col-sm-6 col-xs-12">2</div>
            <div class="col-lg-3 col-md-4 col-sm-6 col-xs-12">3</div>
            <div class="col-lg-3 col-md-4 col-sm-6 col-xs-12">4</div>
        </div>
​
    </div>
 

栅格系统的列嵌套
 

栅格系统内置的栅格系统将内容再次嵌套，简单理解就是一个列内再分成若干小列，我们可以通过添加一个新的.row元素和一系列的.col-sm-元素到已经存在的.col-sm-元素内。
 

在嵌套的时候最好加一个行这样可以取消父盒子的padding值。
 

每个父盒子又可以分为12等份，里面的子盒子也可以指定占父盒子的几份
 

 <div class="container">
        <div class="row">
            <div class="col-md-4">
                  <!-- 在嵌套的时候最好加一个行这样可以取消父盒子的padding值。而且高度自动和父级一样高  -->
                <div class="row row2">
                    <div class="col-md-6 ca">a</div>
                    <div class="col-md-6 cb">b</div>
                </div>
            </div>
            <div class="col-md-4">2</div>
            <div class="col-md-4">3</div>
        </div>
    </div>
 

注意点：
 

1.不能用margin会被挤下来。（原理是100%/盒子所占分数得出盒子的宽度，并且加了float:left;所以加margin就会被挤下来）如果想要每个盒子之间又间距，就要在里面在嵌套一个盒子。
 

列偏移
 

​ 使用col-md-offset-份数，可以将列向右偏移。
 

<div class="container">
        <div class="row">
            <div class="col-md-4">1</div>
            <!-- col-md-offset-份数.是偏移几份的意思. -->
            <div class="col-md-4 col-md-offset-4">2</div>
        </div>
    </div>
 

列排序
 

通过使用.col-md-push-份数（推）和.col-md-pull-份数（拉），类就可以很容易的改变列的顺序。
 

<div class="container">
        <div class="row">
            <div class="col-md-4 col-md-push-8">左</div>
            <div class="col-md-8 col-md-pull-4">右</div>
        </div>
    </div>
