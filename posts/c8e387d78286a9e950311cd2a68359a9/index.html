1、直接部署

（1）拉取openjdk镜像

docker pull openjdk

（2）创建数据卷

docker volume create java_app

 java_app是数据卷名称,可以随便取

（3）将jar文件拷贝到数据卷目录下

将jar 拷贝目录/var/lib/docker/volumes/ java_app/_data/

（4）创建容器

docker run -it -d --network=host --restart=always --name=test_app -v java_app:/usr/src/myapp openjdk java -jar /usr/src/myapp/app.jar

test_app 为容器名称，可以随便取

java_app 为数据卷名称

app.jar 为要运行的 jar 文件

2、dockerfile部署

（1）创建Dockerfile文件

cd  目标文件夹路径
touch Dockerfile

（2）修改Dockerfile文件

vim Dockerfile

按insert键，进入编辑模式,编写内容

## 基础镜像java
FROM openjdk
## 作者是drgaon
MAINTAINER dragon
## 就是你上传的jar包的名称。给jar包起个别名
ADD app_v1.jar app.jar
## 就是在容器中以多少端口号运行
EXPOSE 8080
## 容器启动之后执行的命令，java -jar app.jar 即启动jar
ENTRYPOINT ["java","-jar","app.jar"]

按esc退出编辑，按:wq退出并保存

（3）构建镜像

docker build -t myapp .

注意最后的 . 表示Dockerfile文件在当前目录下
myapp构建之后镜像名称 可以随便起

（4）创建容器

docker run -d --restart=always --name demo -p 8080:8080 myapp

--restart=always 这个表示docker容器在停止或服务器开机后会自动重新启动

demo 是容器名称

myapp 是镜像名称

如果docker run的时候没有加 --restart=always ，然后启动了怎么办？

docker update --restart=always demo

demo 容器名称

3、查看容器

docker ps

查看现在运行的容器

docker ps  -a

查看全部容器
