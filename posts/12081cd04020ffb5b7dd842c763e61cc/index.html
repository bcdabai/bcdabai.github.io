使用 VMware 安装 Linux

VMware 介绍

VMware 是一种常用的、功能强大的虚拟机管理软件，可以模拟出多台计算机，实现在一台机器上同时运行多个独立的操作系统。
VMware 的主要特点：

  • 同一台机器上可以同时运行多个操作系统；
  • 本机系统可以与虚拟机系统进行网络通信；
  • 可以随时修改虚拟机系统的硬件环境。

安装 Linux 操作系统

1.选择好镜像文件后，开启虚拟机，相当于打开电脑的电源启动。
开机同时按 Ctrl + G 或在虚拟机内单击，可使输入定向到虚拟机，然后迅速按 F2，可以打开 Linux 系统的 BIOS 界面。因为这里用光盘安装 Linux 操作系统，BIOS 默认第一启动项是硬盘启动。如果在真实机中通过光盘安装 Linux 操作系统，就需要在安装时，更改第一启动项为光盘启动。安装完成后，再改为硬盘启动。但是，在虚拟机中安装 Linux 操作系统时，无需更改 BIOS 启动项设置，虚拟机会自动识别和调整。
2.开启后，可看到虚拟机的光盘设置，会自动勾选”已连接“和”启动时连接“，如下：

3.如果光盘镜像文件被正确加载和识别的话，会出现光盘启动界面。若报错，就重启该虚拟机，并保证光盘设置正确。

  • Install or upgrade an existing system：安装或升级系统（安装系统时就是用这个）；
  • Install system with basic video driver：采用基本的显卡驱动安装系统；
  • Rescue installed system：修复已安装的系统；
  • Boot from local drive：退出安装，从硬盘启动；
  • Memory test：存储介质检测。

4.直接回车，开始安装。询问是否需要对光盘镜像文件进行检测？由于下载的 CentOS 系统镜像文件本身没什么问题，这里选择”Skip“，加快安装。
5.一切正常的话，会进入图形安装界面，Next → 中文（简体） → Next → 美国英语式 → 下一步 → 基本存储设备 → 是，忽略所有数据（由于初次安装，硬盘空的，不存在数据） → 设置主机名，默认即可 → 下一步 → 亚洲/上海 → 下一步 → 设置 root 账号（超级管理员）的密码 → 下一步
6.询问进行哪种类型的安装？默认为”替换现有 Linux 系统“，表示删除之前的 Linux 分区，然后自动进行分区和格式化。由于在学习安装 Linux 系统，也可以顺便学下如何手动分区，选择”创建自定义布局“，来手动对虚拟硬盘分区：

7.这里介绍如何手动分区和格式化（如果上一步没有选择，可跳过）。磁盘分区过程中，必须有的是根分区（挂载点为 /）和交换分区（没有挂载点），推荐有启动分区（挂载点为 /boot）。一般按照启动分区、交换分区和根分区的顺序操作，如下：
启动分区：创建 → 标准分区 → 创建 → 大小分配 200M → 确定；
交换分区：1024M；根分区：使用全部的可用空间。格式化 → 默认下一步
如果需要给其他子目录（如 /home）单独分配磁盘空间，就在根分区之前操作即可。
8.分区和格式化完成后，出现选择 CentOS 安装选项界面。一般若对 Linux 熟悉，推荐使用”Minimal“最小化安装。初学者选择”Minimal“会导致常用的软件或命令不存在，最好选择”Basic server“。

介绍一下这 8个安装选项：

  • Desktop：桌面，即安装图形界面。
  • Minimal Desktop：最小化桌面，即安装最小化的图形界面。
  • Minimal：最小化安装，没有图形界面。
  • Basic Server：基本的服务器。
  • Database Server：数据库服务器。
  • Web Server：Web服务器。
  • Virtual Host：虚拟主机。
  • Software Development Workstation：软件开发工作站。

不同的安装选项，Linux 系统安装完成后，系统所包含的软件或命令也会有所不同。（如果对 Linux 系统非常熟悉，可以选择”现在自定义“，手段选择要安装的软件包）
9.安装完成后，出现如下界面，并提示重启，点击”重新引导“即可。

重启即可输入账号密码登录，整个 Linux 的操作系统就安装完成了。

VMware 快照功能

VMware 的快照功能可以对虚拟机的某种状态进行一次状态保存，以便日后对这种状态的快速恢复。注意，快照功能只是 VMware 的功能，真实的 Linux 系统并不具备该功能。如果用的不是虚拟机中的 Linux 系统，就要注意对 Linux 系统的相关文件进行备份。

VMware 克隆功能

VMware 的克隆功能可以迅速克隆出当前虚拟机的一个副本，而且无需占用较多的磁盘空间，非常方便和实用。
例如：需要多台 Linux 系统做服务器集群测试时，可以克隆，而且只需占用几十兆的磁盘空间（需采用链接克隆的方式）。
注意：克隆出来的虚拟机只是原始虚拟机的一个副本，如果原始虚拟机被删除了，副本也就无法正常使用了。
右击主界面的虚拟机（如 CentOS6） → 管理 → 克隆 → 下一步 → 虚拟机中的当前状态 → 创建链接克隆 → 设置新虚拟机的名称和保存位置 → 完成。
